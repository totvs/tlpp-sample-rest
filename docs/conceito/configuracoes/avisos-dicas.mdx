## Path parameters

O nome da variável em `:param` é fixado pela **primeira rota registrada** no mesmo path base. Rotas subsequentes devem reutilizar o mesmo nome.

```tlpp
@Get("users/:id")     // registra :id
@Get("users/:userId") // ERRADO — nome diferente no mesmo path base
```

## Annotation vs metadados OpenAPI

`statusCode` em `responses` na annotation é **descritivo** para a documentação. O status HTTP real depende de `oRest:setStatusResponse()` no código.

Roteamento fica nesta seção; enriquecer a spec OpenAPI é trilha [Doc Generate](/docs/tlpp/rest/metadados/visao-geral).
