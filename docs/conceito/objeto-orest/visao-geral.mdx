`oRest` é instanciado **automaticamente** por thread — não construa manualmente com `New()`.

Cada requisição recebe um objeto isolado para leitura de entrada e montagem da resposta. Quando a requisição termina, o objeto é descartado.

:::tip Em outras palavras
No [Quickstart](/docs/tlpp/rest/primeiros-passos/quickstart) você usou `oRest:setResponse()` sem explicar o objeto. Pense no `oRest` como o **mensageiro** entre o HTTP e o seu código: ele traz o que o cliente enviou e leva de volta o que você quer responder.
:::

## Ciclo de uma requisição

1. **Cliente** envia HTTP — ex.: `GET /api/users/1`.
2. **AppServer** recebe na porta REST, aloca uma thread do pool e injeta o `oRest` nessa thread.
3. **Sua função** (`@Get`, handler de `LoadURNs`, etc.) executa — lê entrada com `oRest:getPathParamsRequest()`, `getQueryRequest()`, etc.
4. **Sua função** monta a saída — `oRest:setResponse()`, `setStatusResponse()`, headers.
5. **AppServer** devolve HTTP ao cliente (status + body + headers).
6. **Thread** encerra; o `oRest` daquela requisição é descartado (objeto novo na próxima chamada).
